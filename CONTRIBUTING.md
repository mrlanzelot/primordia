# Contributing

## Workflow

1. Read `README.md` and `docs/FILES.md`.
2. Check active scope in `HEARTBEAT.md`.
3. Implement focused changes in small batches.
4. Update docs when behavior or ownership changes.
5. Run build checks before opening a PR.

## Local Commands

```bash
make install
make dev
make build
```

## Expectations

- Keep backend systems clear and separated by purpose.
- Keep frontend rendering loop fast and typed.
- Avoid stale scaffolding and unused dependencies.
- Keep `.agent` planning files synchronized with implementation.
